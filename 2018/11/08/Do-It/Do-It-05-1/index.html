1. 함수란?

함수

  • 프로그램의 코드를 저장하는 공간
  • 코드를 메모리에 저장했다가 필요할 때 마다 호출해 사용
  • [EX] TV 리모컨
    • 코드 저장: 각 채널의 버튼: 해당 채널로 바꿀 수 있음
    • 함수 호출: 리모컨의 버튼 클릭

변수와 함수의 차이점



2. 기본 함수 정의문

함수 정의문

  • 함수를 사용해 코드를 저장한 것

  • 함수에서는 function 키워드를 사용해 변수 선언

  • 함수 정의문 기본형


Hoisting: 일반 함수 정의 방식과 함수 선언 참조 방식의 방식의 차이점

  • 호이스팅(Hoisting) : 물건을 끌어올리다.
    • 일반 함수 정의: 함수 호출 시 호이스팅(Hoisting) 기술 지원
    • 익명 함수 선언 참조 방식: 호이스팅 지원 X

  • 호이스팅 적용

    • 함수 정의문보다 호출문이 먼저 나와도 함수 정의문을 끌어올려 함수 호출



  • Hoisting 실습 예제



  • 함수 정의문 실습 예제
    • [배경색 바꾸기] 버튼을 클릭할 때마다 배경색이 바뀌도록 하는 예제
    • getElementById
      • id 값을 이용해 문서 객체(태그)를 선택하는 메서드
      • CSS의 id 선택자와 비슷한 역할

    • 문서객체.style.스타일속성 = 새 값;

      • 선택한 문서 객체에 스타일 적용




3. 매개변수가 있는 정의문
  • 매개변수 : parameter
  • 기본 함수 정의문의 문제점 해결
    • 문제점: 함수를 호출할 때 값을 전달할 수 없음
    • 단순히 함수명으로 호출해서 코드 실행

  • 매개변수가 있는 함수 정의문

    • 함수 호출 시, 전달하고자 하는 값을 입력해 호출 가능
    • 전달된 값을 매개변수가 받아 함수 정의문에서 사용 가능

  • 매개변수가 있는 함수 정의문 기본형
    • 함수 호출 시, 데이터1은 매개변수1에 저장되고, 데이터2는 매개변수2에 저장된다.


  • 매개변수 함수 실습 예제 1

    • 함수 호출 시, 이름과 사는 지역의 데이터를 함수 정의문의 매개변수 name과 area에 각각 전달하여 함수 안에 있는 실행문의 매개변수에 저장된 데이터를 불러와 출력하는 예제



  • 매개변수 함수 실습 예제 2

    • 질의응답 창(prompt)을 통해 방문자의 아이디와 비밀번호를 입력받기

      • 잘못된 아이디: “존재하지 않는 아이디입니다” 라는 경고창 팝업
      • 잘못된 비밀번호: “잘못된 비밀번호입니다.” 라는 경고창 팝업
      • 아이디와 비밀번호 일치: 환영 문구 출력




4. 매개변수 없이 함수에 전달된 값 받아오기
  • arguments : 매개변수가 없는 함수 정의문에서 참조하는 배열 변수
  • 함수 정의문에서 매개변수를 사용하는 것처럼 함수 호출문의 값 받아오기

  • 함수 정의문의 매개변수가 없는 상태에서 데이터를 전달하여 함수를 호출하면 그 값이 배열에 저장되고, 그 배열을 참조하는 변수를 arguments라고 한다.

  • arguments 라는 배열 변수가 자동생성 되어 해당 함수 안에서 사용 가능

  • arguments 기본형
    • 함수 호출 시, 데이터1은 매개변수1에 저장되고, 데이터2는 매개변수2에 저장된다.


  • arguments 실습 예제
    • 매개변수를 생략한 함수 호출문에서 3개의 숫자형 데이터를 전달
      • 배열에 저장되는 숫자형 데이터는 arguments 변수로 참조 가능
      • arguments 인덱스를 이용해 배열에 저장된 값을 불러와 합을 구하여 결괏값(60)을 화면에 출력

    • for 문을 이용해 전달받은 데이터의 갯수만큼 반복해 sum에 값을 더하는 것도 가능
