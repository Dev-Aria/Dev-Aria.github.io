1. MyBatis 란?

1) 정의
  • 객체지향언어인 자바의 관계형 데이터베이스 프로그래밍을 쉽게 도와주는 프레임워크
  • JDBC 를 보다 편하게 사용하기 위해 개발된 것
    • JDBC(Java Database Connectivity)
      • 자바 프로그램이 데이터베이스와 연결되어 데이터를 주고 받을 수 있게 해주는 프로그래밍 인터페이스
    • 다양한 관계형 데이터베이스 프로그래밍을 위해 API 제공

2) 특징
  • SQL문이 코드로부터 완전히 분리 : 기존에는 DAO 파일에 모든 SQL문을 작성했지만, Mapper 파일에 SQL 코드를 입력해 놓고, DAO 파일에서 필요할 때마다 가져와서 사용 가능
  • 생산성 : 코드가 짧아진다.
  • 유지보수성 향상 : Mapper에서만 SQL코드를 유지보수하므로, DAO에는 아무 영향이 없다.

3) 구성
  • MyBatis 환경설정 파일(SqlSessionConfig.xml) : MyBatis가 JDBC 코드를 실행하는 데 필요한 전반적인 세팅
    • TypAlias 설정 : 사용할 모델 클래스에 대한 별칭 설정
    • DB 연동을 위한 설정 : Database에 어떻게 접속할 것인지에 대한 설정(environment)
    • Mapper 설정 파일 등록 : 매핑 설정이 어디있는지(mapper)
  • Mapper 설정 파일(SQL.xml) : SQL문과 관련된 설정을 하는 파일로서, MyBatis 설정팡리(SqlSessionConfig.xml)에 등록해야 함
    • 주요 구성 요소
      • SQL문 등록 태그
        • SQL문 태그의 구성요소 : Parameter, Result, SQL문 등록
        • SQL 태그 : insert, delete, update, select
        • 공통 SQL문 설정 태그 : sql
      • select 결과 처리 설정
        • resultMap


2. context-datasource.xml

  • resources/egovframework/spring/
  • 연결할 Database의 접속 정보 를 담는 파일

  • MySQL 양식에 맞춰 입력



3. context-mapper.xml

  • resources/egovframework/spring/
  • 업무폴더별로 Mapper의 경로가 다르므로 MapperConfigurer를 담당하는 bean을 수정해야 한다.

  • basePackage 프로퍼티의 value는 Mapper파일의 경로를 담는다. 이때, 어떤 업무폴더의 Mapper도 인식할 수 있어야 하므로, 업무폴더 위치를 sample에서 **로 수정한다.




4. SQL.xml

  • resources/egovframework/sqlmap/examples/mappers/
  • mapper 태그에 해당 업무폴더의 Mapper.java 파일 경로 지정

  • Controller에서 Mapper까지 같은 이름으로 이어온 xml파일쿼리문id에 해당하는 SQL 태그 활용



5. sql-mapper-config.xml

  • resources/egovframework/sqlmap/examples/
  • 테이블 칼럼에 접근할 때 사용하는 VO를 SQL.xml에서도 사용하려면 이 파일에 미리 정의를 해둬야 한다.

  • VO가 아니더라도 쿼리문에서 활용하고자 하는 클래스는 미리 정의해둬야 한다.
