1. Annotation 이란?

1) 정의
  • @를 이용한 주석
  • 자바코드에 주석을 달아 특별한 의미를 부여한 것
  • 컴파일러가 특정 오류를 억제하도록 지시하는 것과 동일한 기능
  • 프로그램 코드의 일부가 아닌 프로그램에 관한 데이터를 제공
  • 코드에 정보를 추가하는 정형화된 기법

2) 용도
  • 컴파일러를 위한 정보를 제공하기 위한 용도
    • @Override
  • 런타임의 리플렉션을 이용해 특수 기능을 추가하기 위한 용도
    • @Controller
    • 자바 리플렉션 : 다른 언어에는 존재하지 않는 기능. 컴파일 시간이 아닌 실행시간에 동적으로 특정 클래스의 정보를 객체를 통해 분석 및 추출해내는 프로그래밍 기법
  • 컴파일 과정에서 어노테이션 정보로부터 코드를 생성하기 위한 용도

3) 기본 어노테이션(JDK에서 제공)
  • @Override : 해당 메소드가 부모 클래스에 있는 메소드를 제정의했다는 것을 명시적으로 선언
  • @deprecated : 더 이상 사용되지 않는 클래스나 메소드 앞에 추가
  • @SuppressWarnings : 프로그램에 문제가 없음에도 컴파일러가 Warnning을 띄울 경우, 프로그래머에게 무시하라고 알림

4) 어노테이션 사용법

  • 메소드, 필드 등 어디에서나 사용 가능



2. MVC패턴 이란?

1) 정의
  • Model, View, Controller 의 약자
  • 소프트웨어 공학에서 사용되는 소프트웨어 디자인패턴
  • 디자인패턴 : 프로그램을 개발하는 중에 발생했던 문제점들을 정리하여 상황에 따라 간편하게 적용해서 쓸 수 있는 것을 정리해 쉽게 쓸 수 있는 형태로 만든 “규약”
  • Model : 백그라운드에서 동작하는 로직을 처리, 웹페이지에 필요한 기능(Java 소스)
  • View : 사용자가 보게 될 결과 화면을 출력, 실질적으로 보여지는 페이지(JSP)
  • Controller : 사용자의 입력처리와 흐름 제어를 담당, View와 Model을 연결시켜주는 역할
  • Model1 : JSP에서 출력과 로직을 전부 처리
  • Model2 : JSP에서 출력만 처리

2) Model 1
  • JSP가 사용자의 요청을 전부 다 처리
  • 웹 브라우저 사용자의 요청을 받은 JSP는 JavaBean이나 Service클래스를 사용해 웹브라우저가 요청한 적업을 처리하고 그 결과를 출력


3) Model 2
  • 웹브라우저 사용자의 요청을 서블릿이 받음
  • 서블릿은 웹브라우저의 요청을 받아 View로 보여줄 것인지 Model로 보내줄 것인지 정하여 전송
  • View페이지는 사용자에게 보여주는 역할만 담당하고, 실질적인 기능의 부분은 Model에서 담당

  • 실질적으로 보여지는 HTML과 JAVA 소스를 분리해 놓았기 때문에 모델1 방식에 비해 개발확장과 유지보수에 용이




3. 업무폴더 생성

  • java 와 jsp 양 쪽에 생성해야 한다.
  • java/egovframework/example
  • webapp/WEB-INF/jsp/egovframework/example
  • 업무폴더는 관련 페이지들을 기준으로 나누는 것이 좋다.
  • 개인적으로 Table을 기준으로 나누는 편이다.
  • MySQL에 생성한 Table이 member와 drama이므로 각각의 업무폴더를 생성한다.

  • Controller - Service - ServiceImpl - Mapper



4. Controller

  • java/egovframework/example/업무폴더/web

  • 주의 : Contoller - Service - ServiceImpl - Mapper 순으로 연결되므로, 클래스명, 변수명, 메서드명은 계속 일치해야한다.

6. ServiceImpl
