02. eGovFrame 프로젝트에 환경설정하기

1. Maven
2. Tiles 나누기




1. Maven

Maven : jar 파일을 관리하는 곳

  • jar 파일: 전자정부프레임워크를 돌리기 위한 자바소스만 압축한 파일
  • Package Explore에서 Maven Dependencies를 통해 현재 받은 jar파일들을 확인 가능(우유병 모양)
  • pom.xml을 통해 jar 파일 추가/삭제 관리
  • 인터넷 연결 필수
    • pom.xml의 dependency 정보를 조합해 해당 웹사이트에서 jar파일을 다운받기 때문


2. Tiles 나누기

1) Tiles
  • 공통적인 부분을 따로 빼서 전체 적용해주기 위해 필요한 기법
  • 수 많은 include 대신 간결하게 대체 가능
    • 웹 페이지의 header, body, left bar, footer 등의 구분에 용이

2) Tiles 설정 방법

① pom.xml 의 properties에 org.apache.tiles-version 2.2.2 추가

② dependency 추가

  • 4개의 jar파일 등록

    ③ dispatcher-servlet.xml에서 설정하기

    • 뷰 컨트롤러
    • 파일 위치 : webapp/WEB-INF/config/egovframework/springmvc/
    • 이 파일은 AA(Application Architecture) 영역으로, 아주 중요한 파일

    • ViewResolver 설정

      • 기존의 .jsp 순서를 order=”2” 로 설정

      • tilesViewResolver의 순서를 order=”1” 로 설정

    ④ tiles 폴더 생성 후 default-layout.xml 파일 만들기

    • dispatcher-servlet.xml에서 지정한 경로에 파일 생성
      • /WEB-INF/tiles/default-layout.xml
    • 코드 분석
      • 자주 바뀌는 content의 경로는 따로 분리해 상속(extends) 후 재정의
      • “/.tiles”는 “어떤폴더/어떤파일” 이던지 .tiles만 붙이면 적용시킨다는 의미
      • “/WEB-INF/jsp/egovframework/example/{1}/{2}.jsp”
        • ~example 폴더 안에 어떤폴더/어떤파일이 오던 .jsp 파일들을 다 적용해준다는 의미
        • 영역을 잡아준다
      • 메인, 로그인, 게시판 등 레이아웃이 바뀔 때는 위의 세팅처럼 추가하여 관리

    ⑤ tiles에 지정한 경로에 파일 생성

    • “/WEB-INF/jsp/egovframework/example/cmmn/default-layouts.jsp”
    • “/WEB-INF/jsp/egovframework/example/cmmn/default-header.jsp”
    • “/WEB-INF/jsp/egovframework/example/cmmn/default-left.jsp”
    • “/WEB-INF/jsp/egovframework/example/cmmn/default-nav.jsp”
    • “/WEB-INF/jsp/egovframework/example/cmmn/default-footer.jsp”

    ⑥ index.jsp 수정

    ⑦ java의 main 업무폴더 생성

    • src/main/java/egovframework/example/main/web/MainController.java 생성
    • index.jsp에서 설정한대로 main.do가 가장 먼저 실행되게 하기 위함

    ⑧ jsp의 main 업무폴더 생성

    • src/main/webapp/WEB-INF/jsp/egovframework/example/main/main.jsp 생성
    • MainController.java의 initMain() 메서드의 return 값으로 반환되는 main/main.tiles의 실행을 위함
